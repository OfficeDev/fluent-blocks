# Big message

A big centered title message, with opportunities for an illustration above, body text below the title, and a group of up to three actions.

Designed primarily for error messages, empty states, success messages, etc.

### Welcome Screens

For a welcome screen on a first-run experience, use an inviting illustration or branding. It should give users a sense of what your app does and offer first steps for getting started.

## Empty state

An empty state should provide clear direction on what the user can do to begin working with the host feature. Sometimes there may not be an action for the user to take, in which case actions are not required but text should clearly explain what the user can expect from the page in the future.

### With actions

<Canvas>
  <Story name="Empty with Actions">
    <FluentPatternsProvider>
      <BigMessage
        media={{ illustration: 'empty' }}
        actions={{
          primary: {
            actionId: 'a1',
            label: 'Create a task',
            icon: 'add',
          },
          secondary: {
            actionId: 'a2',
            label: 'Find an existing task',
            icon: 'search',
          },
        }}
        viewportHeight={false}
      />
    </FluentPatternsProvider>
  </Story>
</Canvas>

### Without actions

<Canvas>
  <Story name="Empty">

      <BigMessage

        description={[{ text: 'Until then, enjoy your time off.' }]}
        viewportHeight={false}
      />
    </FluentPatternsProvider>
  </Story>
</Canvas>
